JSON Schema DSL

Declare the exact JSON a model must return. Blueprint compiles to strict JSON Schema, so you never hand-write required, additionalProperties, or provider wrappers.

The Basics

A blueprint describes one output shape: the fields a chat reply must contain, their types, and what each field means. When a thread's output mode is JSON, the model is forced to return exactly that shape — no extra keys, no missing keys, no prose.

You write blueprints in the chat's Structured Output editor. Preview shows the compiled JSON Schema, Generate drafts a blueprint from a plain-English description using the thread's model, and the ? button opens a quick-start card. The smallest useful blueprint is a single Return block:

Minimal Blueprint 
Return Answer:
  verdict: Boolean
  reason: String(max=200)  # one short sentence explaining the verdict

Worked Example

One blueprint using most of the language: a reusable Schema, a description comment, a nullable field, an inline nested object, an enum, and a list. Below it is the exact schema the compiler produces.

Blueprint 
Schema Friend:
  name: String(min=1)
  ranking: Float(min=0.0, max=1.0)  # closeness, 1.0 is closest

Return Person:
  name: String(min=1)
  age: Optional[Integer]  # null when unknown
  address:
    street: String
    city: String
  status: Enum["active", "inactive"]
  friends: List[Friend]

Writing Blueprint

Declarations

Every blueprint has exactly one Return, which declares the reply shape. Schema blocks are optional and declare reusable shapes you reference by name.

You Write You Get
Return Name: with indented fields The reply is an object with those fields.
Return Name where Name is a declared Schema The reply is that schema. No duplicate copy ships in $defs.
Return List[Name] or any type expression The reply is that type — see Reuse & Roots for how non-object roots travel.
Schema Name: with indented fields A reusable shape for fields, lists, and unions.

Fields and Indentation

Fields are name: Type, one per line, indented exactly two spaces per nesting level. Field names use letters, digits, and underscores. Tabs, odd indents, and duplicate field names are compile errors.

Comments Are Descriptions

A trailing # comment on a field or declaration line compiles into that node's description — the model reads it, so this is where you say what a field means, its units, or when it should be null. Full-line comments are ignored and never reach the model. A trailing # with no text is a compile error.

Nesting Inline

A bare name: opens a nested object, and name: List: (or name: List(min=1):) opens a list of objects. Children are indented two more spaces. Inline blocks cannot be unioned or wrapped in Optional — declare a named Schema when you need that.

Inline Nesting 
Return Order:
  id: Uuid
  customer:  # buyer details
    name: String(min=1)
    email: Email
  items: List(min=1):
    sku: String
    qty: Integer(min=1)

Nullable Fields

Optional[Integer] and Integer | None mean the same thing: the key is always present, and its value may be null. Strict schemas have no way to say "this key may be missing" — every declared field appears in every reply.

Types

Primitives

  • String — text.
  • Integer — whole number.
  • Float — decimal number.
  • Boolean — true or false.
  • None — null; used in unions like Integer | None.

Containers

  • List[T] — an array of any type, including schemas.
  • Enum[...] — a fixed set of literals, all one type: Enum["draft", "final"], Enum[1, 2, 3], or Enum[True, False].
  • Optional[T] — shorthand for T | None.
  • A | B — the value is one of the listed types. Empty or duplicate branches are errors.
  • A declared Schema name used as a type, like friends: List[Friend], becomes a $ref.

Formatted Strings

These compile to type: "string" plus a JSON Schema format, which is also a strong hint to the model.

Type Format Example Value
Date date 2026-07-02
Time time 09:30:00
Datetime date-time 2026-07-02T09:30:00Z
Duration duration PT1H30M
Email email ada@example.com
Hostname hostname api.example.com
IPv4 ipv4 192.168.0.1
IPv6 ipv6 2001:db8::1
Uuid uuid 0f8fad5b-d9cb-469f-a165-70867728950e

Constraints

Constraints are optional key=value arguments on a type. min and max adapt to context: character length on strings, numeric value on numbers, item count on lists.

Blueprint Compiles To
String(min=1, max=80) minLength, maxLength
Integer(min=0, max=10) minimum, maximum
Float(exclusive_min=0.0, exclusive_max=1.0) exclusiveMinimum, exclusiveMaximum
Integer(multiple_of=5) multipleOf
List[String](min=1, max=10) minItems, maxItems

There is no regex. When a format type doesn't fit, constrain the length and explain the expected shape in a trailing # description.

Reuse & Roots

Using a Schema name as a type compiles to a $ref into $defs, so the shape is defined once no matter how many fields use it. Schemas may reference each other and themselves — recursive shapes like trees work. Only schemas actually reachable from the Return ship in $defs; anything unused is pruned.

Non-Object Roots

Return List[Person] declares that the whole reply is an array. Providers require the schema root to be an object, so the runtime transports non-object roots inside a single-key wrapper object and unwraps the reply before you see it. You always get the shape you declared; the wrapper is invisible.

Root Array 
Schema Person:
  name: String

Return List[Person]

What the Compiler Guarantees

  • Every field is required. The model cannot omit keys; use Optional[T] for values that may be null.
  • Objects are closed. Every compiled object gets additionalProperties: false — no surprise keys.
  • Errors aggregate. One compile pass reports every error it finds, each prefixed with its line number.
  • Syntax is strict. Two-space indentation, no tabs, no trailing commas, no empty or duplicate union branches. Common slips get hints, like str → "Did you mean String?".
  • Provider limits are checked locally. At most 5,000 properties, 10 nesting levels, 1,000 enum values, and 120,000 schema characters — mirroring OpenAI strict mode, so a blueprint that compiles here is accepted there.
  • The provider wrapper is internal. The runtime adds strict, the response name, and the format envelope. Blueprints never contain provider boilerplate.
Provider Wrapper, Added by the Runtime 
{
  "type": "json_schema",
  "name": "person",
  "strict": true,
  "schema": {
    "...": "compiled schema"
  }
}

Errors

The compiler reports every error in one pass, each with a line number, so a broken blueprint is usually fixable in a single edit. The common families:

Error What Happened Example
Unknown type The type isn't built in and isn't a declared Schema. Python names get a hint. phone: str
Missing or extra Return A blueprint needs exactly one Return. Two Return blocks
Unsatisfiable constraint The bounds can never both hold. String(min=5, max=2)
Unsupported argument That type doesn't take that argument. Boolean(min=1)
Bad indentation A line isn't indented in two-space steps, or doesn't match its block level. A field indented three spaces
Sloppy syntax Empty or duplicate union branches, extra commas, or a space before [. String | String
Empty trailing comment A trailing # with no text — write the description or remove it. name: String #
Invalid inline block Only a bare field name or List(...) may open an indented block. address: Optional:
Provider limit The compiled schema exceeds a strict-mode size limit. Over 5,000 properties
